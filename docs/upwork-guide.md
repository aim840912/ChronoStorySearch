## 什麼是 Upwork

Upwork 是全球最大的自由接案平台之一，連接：
- **Freelancer（接案者）**：提供技能服務
- **Client（發案者）**：尋找人才完成專案

### 平台特色

| 特色 | 說明 |
|------|------|
| 全球市場 | 客戶來自世界各地 |
| 安全付款 | 平台託管付款，保護雙方 |
| 評價系統 | 建立信譽和口碑 |
| 多元工作 | 程式開發、設計、寫作、翻譯等 |

## 註冊與設定個人檔案

### 步驟 1：註冊帳號

1. 前往 [upwork.com](https://www.upwork.com)
2. 點擊 "Sign Up"
3. 選擇 "I want to work"（我要接案）
4. 填寫基本資料

### 步驟 2：完善個人檔案（非常重要！）

#### 必填項目

| 項目 | 建議 |
|------|------|
| **Profile Photo** | 專業頭像，清晰的正面照 |
| **Title** | 簡潔有力，例如：`Full-Stack Developer | React & Node.js Expert` |
| **Overview** | 300-500 字的自我介紹 |
| **Hourly Rate** | 新手建議 $15-30/hr 起步 |
| **Skills** | 加入 5-10 個相關技能標籤 |
| **Portfolio** | 展示 3-5 個作品 |

#### Overview 範例

```
Hi, I'm Tien, a Full-Stack Developer based in Taiwan with expertise in:

✅ Frontend: React, Next.js, TypeScript, Tailwind CSS
✅ Backend: Node.js, Express, PostgreSQL
✅ Cloud: Vercel, AWS, Supabase

What I offer:
- Clean, maintainable code
- Clear communication
- On-time delivery

I'm passionate about building user-friendly web applications and always eager to learn new technologies.

Let's discuss your project!
```

## Upwork 的運作方式

### 工作流程

```
1. 客戶發布工作 (Job Post)
        ↓
2. 你投遞提案 (Submit Proposal)
        ↓
3. 客戶審核提案
        ↓
4. 面試/討論 (Interview)
        ↓
5. 客戶發送合約 (Contract)
        ↓
6. 你接受合約，開始工作
        ↓
7. 完成工作，提交成果
        ↓
8. 客戶付款 + 評價
```

## 如何找工作

### 方法 1：搜尋工作

1. 點擊 "Find Work"
2. 使用關鍵字搜尋，例如：
   - `React developer`
   - `Next.js`
   - `TypeScript`
3. 使用篩選器：
   - Experience Level（經驗等級）
   - Budget（預算）
   - Client History（客戶歷史）

### 方法 2：設定 Job Alerts

1. 搜尋你想要的工作類型
2. 點擊 "Save Search"
3. 開啟 Email 通知

### 選擇好工作的標準

| 指標 | 好的跡象 | 警示信號 |
|------|---------|---------|
| **客戶評價** | 4.5+ 星 | 無評價或低於 4 星 |
| **付款歷史** | $1000+ 已付款 | $0 付款記錄 |
| **工作描述** | 清楚具體 | 模糊不清 |
| **預算** | 合理範圍 | 過低（如 $5 做整個網站） |
| **驗證狀態** | Payment verified ✓ | 未驗證 |

## 提案（Proposal）撰寫技巧

### 提案結構

```
1. 開場：展示你理解客戶需求
2. 中段：說明你如何解決問題
3. 結尾：行動呼籲（CTA）
```

### 提案範例

```
Hi [Client Name],

I read your job post carefully and understand you need a React developer
to build a dashboard with real-time data visualization.

I have 3+ years of experience with React and have built similar dashboards
before. Here's a relevant project: [link to portfolio]

For your project, I would:
1. Set up the React/Next.js framework
2. Implement the data visualization using Chart.js
3. Add real-time updates with WebSocket
4. Ensure responsive design for all devices

I can start immediately and deliver within your timeline.

Would you be available for a quick call to discuss the details?

Best,
Tien
```

### 提案 Do's and Don'ts

| ✅ Do | ❌ Don't |
|------|---------|
| 客製化每份提案 | 複製貼上相同內容 |
| 提及具體細節 | 寫太籠統的內容 |
| 展示相關作品 | 只說「我可以做」 |
| 問有意義的問題 | 問工作描述已有的資訊 |
| 保持簡潔 | 寫太長的提案 |

## Connects 系統

### 什麼是 Connects？

Connects 是 Upwork 的「貨幣」，用來投遞提案：

| 工作類型 | 所需 Connects |
|---------|--------------|
| 一般工作 | 2-6 Connects |
| 熱門工作 | 可能更多 |

### 如何獲得 Connects

| 方式 | 數量 |
|------|------|
| 每月免費獲得 | 10 Connects |
| 購買 | $0.15/Connect |
| 被雇用（合約開始） | 退還該提案的 Connects |

### 省 Connects 技巧

1. **選擇性投遞** - 只投符合你能力的工作
2. **優先新發布** - 競爭較少
3. **看客戶歷史** - 有雇用記錄的客戶更可能回覆

## 收費與付款

### Upwork 服務費

| 累計收入（同一客戶） | 服務費 |
|---------------------|--------|
| 首 $500 | 20% |
| $500.01 - $10,000 | 10% |
| $10,000.01+ | 5% |

### 範例計算

```
客戶付款：$100
Upwork 服務費（20%）：$20
你實際收到：$80
```

### 付款方式

1. **Direct to Local Bank**（推薦）
2. PayPal
3. Payoneer
4. Wire Transfer

### 付款時間

- **每週付款**（Weekly）：每週三處理
- **固定價格**：里程碑完成後付款

## 合約類型

### 1. Hourly（時薪制）

| 特點 | 說明 |
|------|------|
| 計費方式 | 按工作時數 |
| 時間追蹤 | 使用 Upwork 的 Time Tracker |
| 付款保障 | 有記錄的時間自動付款 |
| 適合 | 長期合作、需求不明確的專案 |

#### Time Tracker 使用

1. 下載 Upwork Desktop App
2. 選擇合約
3. 點擊 "Start" 開始計時
4. 每 10 分鐘截圖一次（證明你在工作）
5. 點擊 "Stop" 結束計時

### 2. Fixed-Price（固定價格）

| 特點 | 說明 |
|------|------|
| 計費方式 | 整個專案或里程碑 |
| 付款保障 | 客戶先託管（Escrow） |
| 適合 | 需求明確、短期專案 |

#### 里程碑（Milestone）範例

```
專案：建立電商網站
總價：$1000

里程碑 1：UI 設計 - $200
里程碑 2：前端開發 - $400
里程碑 3：後端整合 - $300
里程碑 4：測試與上線 - $100
```

## 溝通與工作流程

### 平台內溝通

1. **Messages** - 即時訊息（類似聊天）
2. **Video Call** - 內建視訊（Zoom 整合）

### 溝通技巧

| 情境 | 建議回覆 |
|------|---------|
| 客戶問進度 | "I've completed X and Y. Currently working on Z. Expected to finish by [date]." |
| 需要更多時間 | "I need 2 more days to ensure quality. Is that okay?" |
| 不確定需求 | "Could you clarify what you mean by X? I want to make sure I understand correctly." |
| 完成工作 | "I've completed the work. Please review and let me know if you need any changes." |

### 工作流程建議

```
1. 收到合約 → 確認需求細節
2. 開始前 → 問清楚不明確的地方
3. 工作中 → 定期更新進度
4. 完成後 → 提交成果 + 詢問回饋
5. 修改後 → 請客戶關閉合約 + 給評價
```

## 新手常見問題

### Q1：沒有評價怎麼辦？

**解決方案**：
1. 先接小案子建立評價
2. 報價稍低吸引客戶
3. 提供詳細的 Portfolio
4. 在提案中強調你的優勢

### Q2：客戶不回覆怎麼辦？

這很正常！平均回覆率約 5-10%。
- 繼續投其他工作
- 不要追問（會顯得急躁）

### Q3：價格怎麼定？

| 經驗 | 建議時薪（USD） |
|------|----------------|
| 新手 | $15-25 |
| 1-2 年 | $25-40 |
| 3-5 年 | $40-70 |
| 專家 | $70+ |

### Q4：要用中文還是英文？

**英文**！Upwork 是國際平台，大部分客戶說英文。

## 安全注意事項

### 🚨 常見詐騙手法

| 詐騙類型 | 特徵 |
|---------|------|
| **假支票詐騙** | 要求你先墊款 |
| **平台外付款** | 要求用其他方式付款 |
| **個資詐騙** | 要求銀行帳號、身分證 |
| **免費試做** | 要求大量免費工作 |

### ✅ 安全守則

1. **永遠在平台內溝通和付款**
2. **不要分享個人敏感資訊**
3. **不要在合約開始前工作**
4. **可疑訊息直接檢舉**

### 辨識好客戶

```
✅ Payment verified（付款已驗證）
✅ 有付款歷史（$1000+）
✅ 有雇用歷史（10+ hires）
✅ 評價 4.5+ 星
✅ 工作描述清楚具體
```
